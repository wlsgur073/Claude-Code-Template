# ステップ

## Step 1: 引数パース

`$ARGUMENTS` を読み取り。`--fix` が含まれている場合、リントは自動修正を有効にして実行。

## Step 2: ビルド

`npm run build` を実行して TypeScript がエラーなしでコンパイルされることを確認。
ビルドが失敗した場合、エラーを報告して停止 — リントやテストには進まない。

## Step 3: リント

`npm run lint`（`--fix` が指定された場合は `npm run lint -- --fix`）を実行。
残りの警告やエラーを報告。

## Step 4: テスト

`npm test` を実行してフルテストスイートを実行。
失敗がある場合、テスト名とエラーメッセージを報告。

## Step 5: サマリー

サマリーテーブルを出力：

| チェック | ステータス |
| -------- | ---------- |
| ビルド | pass / fail |
| リント | pass / fail（N 件の警告） |
| テスト | pass / fail（N/M パス） |

すべてのチェックが通過した場合、プロジェクトがコミット可能であることを確認。
いずれかのチェックが失敗した場合、エラー出力に基づいて最も可能性の高い修正を提案。
