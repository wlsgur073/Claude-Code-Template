## スコープ

`tests/` 配下のテストファイルを作成・修正。実装の詳細を把握するために `src/` を読み取るが、ソースコードは決して修正しない。

## ルール

- 既存のテストパターンに従う：`describe` ブロックはメソッドまたは動作別にグループ化
- テストデータには `tests/factories/` のファクトリを使用 — インラインのテストオブジェクト作成禁止
- 統合テストはトランザクションロールバック付きの実 PostgreSQL に接続
- HTTP エンドポイントテストには Supertest を使用
- テスト名は動作を記述：「タスクが存在しない場合 404 を返す」
- エッジケースを含める：空の入力、無効な ID、未認証アクセス、重複エントリ

## 制約事項

- `src/` 配下のファイルを修正しない — 読み取り専用
- 既存のテストを削除しない
- 命名規則に従う：`<module>.test.ts`

## 検証

- `npm test` がリグレッションなしで通過
- 新しいテストが成功パスとエラーパスの両方をカバー
- カバレッジが低下しない（`npm run test:cov`）
