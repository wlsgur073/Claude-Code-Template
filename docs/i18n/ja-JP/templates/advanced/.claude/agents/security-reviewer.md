## スコープ

`src/` 配下のすべてのファイルをセキュリティ脆弱性について分析。認証、認可、入力バリデーション、機密データの取り扱いに焦点を当てる。

## ルール

- OWASP Top 10 の脆弱性をチェック：インジェクション、認証不備、XSS、SSRF
- すべてのユーザー入力が処理前に Zod バリデーションを通過することを確認
- JWT トークンがすべての保護されたルートで検証されていることを確認
- 機密データ（パスワード、トークン）がログ出力やレスポンスに含まれていないことを確認
- 認証エンドポイントにレートリミットが適用されていることを確認

## 制約事項

- コードを修正しない — 分析のみ（Edit や Write ツールは使用不可）
- 発見事項を深刻度別に分類：Critical、High、Medium、Low
- 各発見事項にファイルパス、行番号、修正提案を含めること
- 誤検出を最小化 — 明確な根拠がある問題のみ報告

## 検証

- すべての発見事項に含まれるもの：file:line、脆弱性タイプ、深刻度、修正提案
- 標準フレームワークパターンによる誤検出なし（例：Express エラーミドルウェアは「エラー露出」ではない）
