# アーキテクチャ

## レイヤー構造

上から下の順序：

1. **API ハンドラ** (`src/api/`) -- HTTP リクエスト/レスポンス、入力バリデーション
2. **サービス** (`src/services/`) -- ビジネスロジック、オーケストレーション
3. **リポジトリ** (`src/repos/`) -- データベースアクセス、SQL クエリ
4. **モデル** (`src/models/`) -- Zod スキーマ、TypeScript インターフェース

## 依存方向

- ハンドラはサービスを呼び出す -- リポジトリやモデルを直接呼び出さない
- サービスはリポジトリを呼び出す -- データベースに直接アクセスしない
- リポジトリは型安全性のためにモデルを使用
- レイヤー間の循環依存禁止
- `src/utils/` の共有ユーティリティはすべてのレイヤーからインポート可能

## モジュール境界

- 各エンティティ（Task、User、Comment）は各レイヤーに固有のファイルを持つ
- エンティティ間の呼び出しはサービスを経由、リポジトリへの直接アクセス禁止
- `src/errors/` ディレクトリは共有インフラ -- すべてのレイヤーからインポート可能
