# テストコンベンション

## フレームワーク
- ユニットテストと統合テストに Jest と ts-jest を使用
- HTTP エンドポイントテストに Supertest を使用

## 構造
- `src/` ディレクトリをミラーリング：`tests/services/` は `src/services/` をテスト
- `describe` ブロックはメソッドまたは動作別にグループ化
- テストファイル命名：`<module>.test.ts`

## データ
- テストデータには `tests/factories/` のファクトリを使用 — インラインのオブジェクト作成禁止
- 各ファクトリはデフォルトで有効なエンティティを返す；テストに必要な部分のみオーバーライド
- 例：`createUser({ email: 'test@example.com' })`

## データベース
- 統合テストは実 PostgreSQL に接続 — データベースをモックしない
- 各テストファイルはスイート終了後にロールバックされる個別トランザクションを使用
- セットアップとティアダウンに `tests/helpers/db.ts` を使用

## カバレッジ
- src/services/ の最低 80% ラインカバレッジ
- すべてのエラーパスに最低 1 つのテスト必須
- `npm run test:cov` で確認
