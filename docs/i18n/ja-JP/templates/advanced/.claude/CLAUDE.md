# プロジェクト指示文（代替配置）

プロジェクトルートの代わりに `.claude/CLAUDE.md` に指示文を配置する方法を示しています。完全な例は `templates/advanced/CLAUDE.md`（ルート配置）を参照してください。

Claude はプロジェクトルートと `.claude/` ディレクトリの両方で `CLAUDE.md` を自動検出します — 追加設定は不要です。両方の場所に指示文を配置しないでください。競合が発生する可能性があります。

## `.claude/CLAUDE.md` を使用する場合
- プロジェクトルートをクリーンに保ち、ドットファイルを減らしたい場合
- ルートに設定ファイルが既に多い場合（eslint、prettier、tsconfig など）
- 他の Claude Code 設定（.claude/rules/、.claude/agents/）と一緒にまとめたい場合

## ルートの `./CLAUDE.md` を使用する場合
- リポジトリ閲覧時にファイルが一目で見えるようにしたい場合
- 設定ファイルが少ない小規模プロジェクトの場合
- 新規コントリビューターのアクセシビリティを最大化したい場合
